1-1. AX 개념 이해와 실습 환경 정의

[안내] 실습 도구로 Microsoft Copilot을 선정한 이유

본 교육의 실습 도구는 Microsoft Copilot을 활용합니다. 사내망 내에서 Gemini를 사용하려면 개별 권한 신청이 필요했으나, 최근 보안 정책 변경으로 인해 개인이 사내망 내 Gemini 권한을 직접 신청하는 것이 제한되었습니다. 또한, 모든 인원이 Gemini Enterprise(유료 라이선스) 권한을 균일하게 가지고 있는 상황도 아닙니다.

반면, Microsoft Copilot은 사내 계정(@lge.com)으로 로그인하면 LG전자 임직원 누구나 제한 없이 즉시 사용 가능한 환경이 제공되므로 모든 교육생이 동일한 실습을 완수할 수 있는 최적의 도구로 선정되었습니다.

💡 꼭 기억하세요! AI는 '껍데기(UI)'보다 '프롬프트 논리'가 본질입니다.

시중에 제공되는 수많은 대형 언어 모델(LLM)들은 서비스 껍데기(UI)와 브랜드 이름만 다를 뿐, 질문의 논리 구조(맥락 설정, 구체적 지시, 제약 사항)는 거의 동일합니다. 본 교육에서 다루는 프롬프트 엔지니어링과 에이전트 연동 논리는 향후 권한을 부여받은 Gemini나 ChatGPT, Claude 등 어떤 다른 AI 환경에도 100% 동일하게 이식하여 적용할 수 있습니다.

※ 아울러, MS Copilot 역시 Outlook 이메일 연동, PDF 자동 요약, Excel 데이터 분석 등 실무에 즉시 활용할 수 있는 강력한 기능들을 갖추고 있어 업무 생산성을 혁신적으로 높여줍니다.

핵심 이론: 베이스 모델 vs AI 에이전트

현재 사내 아웃룩 Copilot 등을 실행하면 내부 모델을 GPT나 Claude로 선택해서 쓸 수 있는 기능이 제공됩니다. 그렇다면 공식 사이트(ChatGPT, Claude.ai)에 직접 접속해 사용하는 것과 Copilot에서 사용하는 것은 어떤 차이가 있을까요?

기본 베이스 모델 (Base Model / LLM)

예: GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro 등

인공지능의 순수한 '두뇌' 엔진입니다. 독자적인 대규모 학습 데이터를 기초로 추론, 요약, 코딩 등의 순수 연산 작업을 처리합니다.

⚠️ 공식 웹사이트 직접 접속의 특징:

  • 해당 모델 본연의 지능과 성능을 가장 빠르게 테스트할 수 있습니다.
  • 사내 시스템(이메일, 파일, 캘린더 등)과 자동으로 연동되지 않아 일일이 복사해서 입력해야 합니다.
  • 개인 계정으로 입력한 데이터는 AI 학습에 쓰여 사내 기밀 유출 리스크가 매우 높습니다.

AI 에이전트 / 시스템 통합형 (Copilot)

예: M365 Copilot, Chat GPT, Perplexity 등

베이스 모델을 두뇌로 삼되, 내 업무 데이터(이메일, 일정, 파일)와 유기적으로 연결되어 동작(Action)을 자동 수행하도록 사내 보안망 내에 구축된 지능형 시스템입니다.

✨ Copilot(에이전트 서비스) 활용의 특징:

  • 사내 보안 확보: 사내 계정으로 로그인되어 입력 내용이 외부 학습에 절대 사용되지 않습니다.
  • 업무 환경 결합: 별도 텍스트 복사 없이 내 아웃룩 메일함, 캘린더 데이터를 AI가 직접 조회하여 요약하거나 보고서로 가공합니다.
  • 보안망 내에서 검증된 GPT/Claude의 능력을 가져와 내 사내 데이터에 직접 안전하게 결합시킵니다.

AX(AI Transformation)는 "도구"가 아니라 "동료"입니다.

과거의 IT가 인간의 명령(Command)을 수행하는 계산기였다면, AX 시대의 AI는 인간과 대화(Conversation)하며 맥락을 이해하고 결과물을 함께 만들어가는 협업 파트너입니다.

보안 및 윤리 의식

사내 기밀 데이터 유출을 방지하고 결과의 정확성을 검증하는가?

1-2. 프롬프트 엔지니어링: 페르소나보다 논리

모델이 똑똑해졌습니다. 이제 "너는 20년 차 마케팅 전문가야"라고 최면을 걸기보다, '어떻게 사고할지(Chain of Thought)' 단계를 설계하는 것이 핵심입니다.

과거의 방식 (Legacy)

  • - 장황한 페르소나: "너는 세계 최고의 전략가이며..." (이제는 모델 기본 지능이 높아져 큰 차이가 없음)
  • - 모호한 지시: "최대한 멋지게 써줘"
  • - 단답형 요구: 과정 없이 결과만 바로 도출 시도

차세대 방식 (Modern CoT)

  • - CoT (Chain of Thought): "단계별로 생각해서 논리를 전개해 줘"
  • - 맥락(Context) 집중: 모델에게 역할극을 시키기보다 현재 상황의 데이터를 정확히 전달
  • - 구조적 제약: "첫째, 둘째... 순서로 분석하고 마지막에 표로 정리해"

핵심 기술: Chain of Thought (생각의 사슬)

최신 모델(Gemini 1.5 Pro, GPT-4o 등)은 스스로 추론할 수 있는 능력이 비약적으로 상승했습니다. 결과만 묻지 말고 '생각하는 단계'를 지시하세요.

Step 1. 관점 정의

"먼저 고객 관점과 기술 관점에서 각각 분석해."

Step 2. 근거 나열

"그 분석을 뒷받침할 구체적인 근거를 데이터에서 찾아."

Step 3. 결론 도출

"두 관점을 종합하여 최종 제안서를 요약해."
MODERN LOGIC PROMPT "다음 신제품 기획 초안을 검토해 줘. 장황한 칭찬이나 페르소나 설정은 생략하고 바로 다음 단계에 따라 추론을 전개해. 1. 먼저 이 기획안의 논리적 모순점 3가지를 찾아내. 2. 각 모순점을 해결하기 위한 대안을 LG전자의 핵심 가치와 연결해 제안해. 3. 이 모든 과정을 거쳐 최종적으로 수정된 기획안의 뼈대를 목차 형태로 보여줘. **단계별로 생각(Step by step)해서 가장 합리적인 답변을 도출해.**"

※ 팁: 모델에게 "천천히 단계별로 생각해 봐"라고 한 문장만 추가해도 정답률이 20% 이상 올라갑니다.

1-3. AI 보안 & 할루시네이션 대처법

AI를 잘못 쓰면 오히려 독이 됩니다

AI가 생성한 내용을 검토 없이 그대로 보고·배포하거나, 기밀 데이터를 외부 AI에 입력하는 순간 개인·조직 모두에게 심각한 리스크가 발생합니다. 이 섹션은 AI를 안전하고 정확하게 쓰는 방법을 다룹니다.

  • 최종 보고서에는 전문가 판단 반영

  • AI 출력물을 검토 없이 그대로 임원 보고
  • AI가 생성한 수치를 원본처럼 인용
  • 고객·임원 앞에서 실시간 AI 답변 그대로 발표
  • 외부 공개 문서에 AI 생성 팩트 무검증 사용

PART B. 할루시네이션(Hallucination) 대처법

할루시네이션이란?

AI가 학습 데이터에 없거나 불확실한 내용을 마치 사실인 것처럼 자신 있게 생성하는 현상입니다. 특히 수치, 날짜, 사람 이름, 통계에서 자주 발생하며, 출력 문장이 매우 자연스럽고 그럴듯해서 발견하기 어렵습니다.

① 유형별 할루시네이션 사례

유형 1: 가짜 통계 & 수치

프롬프트: "글로벌 에어컨 시장 점유율 알려줘"

"2024년 LG전자 점유율은 18.7%로 다이킨(22.1%)에 이어 2위입니다."

⚠ 출처 없이 AI가 만들어낸 그럴듯한 숫자!

유형 2: 존재하지 않는 정책·법규

프롬프트: "개인정보보호법 35조 내용 알려줘"

"35조에 따르면 기업은 연 1회 이상 개인정보 영향평가를 의무 수행해야 합니다."

⚠ 실제 법조문과 다른 내용일 수 있음!

유형 3: 없는 사람·사건

프롬프트: "LG전자 CS 관련 사례 알려줘"

"2023년 박○○ 부사장이 발표한 CS 혁신 전략에 따르면..."

⚠ 실제 존재하지 않는 임원·발표 내용!

유형 4: 날짜·버전 오류

프롬프트: "최신 규정 기준 알려줘"

"2024년 개정된 최신 기준에 따르면 ..."

⚠ AI 학습 데이터 기준일 이후 정보는 부정확!

② 할루시네이션 예방 프롬프트 전략

전략 3

출처 함께 요청하기

자기 검증 요청

"위 내용을 작성한 뒤, 스스로 팩트체크해서 불확실한 부분을 [검증필요] 태그로 표시해줘."

1-4. Copilot 접속 및 실행법

"Copilot = 회사 계정으로 로그인한 AI 채팅 서비스"

ChatGPT처럼 말을 걸면 대답하는데, 사내 보안이 적용된 Microsoft 공식 서비스입니다.
별도 프로그램 설치 없이 웹 브라우저에서 바로 사용합니다.

처음이라면 이 3단계만 기억하세요

1

복사 (Ctrl + C)

이메일 본문, 엑셀 데이터, PDF 내용 등
분석할 대상 텍스트를 복사합니다.

2

붙여넣기 + 지시

Copilot 입력창에 붙여넣고
"이걸 한글로 요약해줘"와 같이 명령합니다.

3

결과 확인 후 보완

2-2. PDF 요약 & 보고서 작성

Copilot에 PDF를 직접 올려서 분석할 수 있습니다

아웃룩 Copilot(Claude 모델) 또는 Microsoft Copilot 채팅창에 PDF 파일을 첨부하면, 문서 내용을 읽고 요약·분석·보고서 초안까지 즉시 생성합니다. 긴 보고서, 정책 문서, 외부 리서치 자료를 빠르게 소화하는 데 매우 효과적입니다.

PDF 연동 및 요약 실행법 (2가지 방법)

방법 A: 아웃룩 & 웹 포털에서 직접 첨부

  1. 아웃룩 또는 웹 포털(m365.cloud.microsoft/chat) 접속
  2. 채팅 입력창 하단의 📎 클립(첨부) 버튼 클릭
  3. 내 PC의 PDF 파일을 선택하여 업로드
  4. 업로드 완료 후 프롬프트를 입력하여 요약 지시

방법 B: Edge 브라우저 드래그 앤 드롭 (추천)

  1. 내 PC 폴더에 있는 PDF 파일을 마우스로 클릭합니다.
  2. 클릭한 상태로 MS Edge 브라우저 화면 안으로 드래그해 놓습니다.
  3. 단축키 Ctrl+Shift+.을 눌러 우측 Copilot 사이드바를 켭니다.
  4. 채팅창에 "현재 페이지를 기반으로 요약해줘"라고 입력합니다.

💡 한 번에 여러 PDF를 분석할 때는 [방법 A]의 다중 첨부를 권장하며, 열려 있는 개별 문서를 즉시 분석할 때는 [방법 B]가 간편합니다.

2-3. Excel 데이터 분석: 수식 제작 및 데이터 연동

실무 상황

글로벌 VOC 현황 데이터에서 상담 접수일(A열)과 처리 완료일(B열) 사이의 실제 처리 소요 일수를 구해야 합니다. 날짜 차이 수식이 헷갈리는 상황에서 Copilot으로 수식을 받고, 카테고리별 CSAT(고객만족도) 데이터를 복사하여 보고서용 통계 요약을 즉석에서 뽑아내고자 합니다.

1

엑셀 시트 열 구조 확인 및 질문 준비

분석하고자 하는 엑셀 파일의 정확한 열 이름과 알파벳(A열, B열 등)을 확인합니다.

2

Copilot Web에 수식 짜기 질문 전송

코파일럿 웹 페이지를 켜고, 엑셀 열 이름을 명시하여 어떤 조건의 수식을 짜야 하는지 설명하고 복사 가능한 수식을 요청합니다.

BEST PROMPT (엑셀 함수 수식 제작) "다음 요청에 맞는 엑셀 함수 수식을 만들어 주고 원리를 쉽게 설명해 줘. [요청 사항] - 현재 내 엑셀 데이터의 A열은 '상담 접수일자' (예: 2026-06-01), B열은 '처리 완료일자' (예: 2026-06-05)가 입력되어 있어. - C열의 각 셀에 '처리 완료일자'에서 '상담 접수일자'를 빼서 순수한 처리 소요 기간(일수)을 숫자로만 보여주는 수식을 만들어 줘. - B열(처리 완료일)이 아직 공란인 경우(미완료 건)에는 \"처리중\"이라는 텍스트를 표시하는 IF 조건도 함께 넣어 줘. - 날짜 서식 오류나 문자 데이터형으로 인한 에러를 방지하는 IFERROR 처리도 포함해 줘. 수식은 바로 복사할 수 있게 해주고, 적용법도 짧게 불렛포인트로 알려줘."

3-1. 즉시 실행하는 MVP 개념

MVP (Minimum Viable Prototype)는 회의실에서 말로만 요구사항을 전달하는 한계를 극복하기 위해, AI를 통해 실제 브라우저에서 동작하는 인터랙티브 샘플을 즉시 생성하여 협업하는 방식입니다.

전통적인 방식 (동상이몽)

  • - 텍스트 기획서만 보고 개발/디자인 진행
  • - 결과물이 완성된 후 생각했던 것과 달라 전면 수정 반복
  • - 소통 오류 및 프로젝트 일정 지연 발생

MVP 프로토타이핑 (시각적 협업)

  • - 아웃룩 Copilot(Claude)을 활용해 5분 만에 실제 작동 HTML 생성
  • - 화면 설계와 동적 인터랙션을 회의 중 실시간 눈으로 확인
  • - 개발팀에 "이 샘플 코드 그대로 구축해주세요"라고 명확한 의사전달

🎯

보고용 슬라이드

키보드로 넘기는 인터랙티브 HTML 발표자료

🔧

신규 시스템·서비스 기획 단계의 UI 프로토타입

3-2. 대시보드 MVP 제작

코딩 지식 없이도 아웃룩 Copilot(Claude 모델)으로 실제 브라우저에서 작동하는 대시보드를 즉시 만들고 회의에서 시연할 수 있습니다.

대시보드 MVP 제작 4단계

  1. 아웃룩 앱 실행: 아웃룩을 열고 좌측 사이드바의 Copilot 버튼을 클릭합니다.
  2. Claude 모델 선택: 코파일럿 창 좌측 상단의 '자동' 버튼을 클릭하여 Claude(클로드)로 변경합니다.
  3. 데이터와 함께 프롬프트 전송: 아래 프롬프트에 분석할 데이터(엑셀 복사본 등)를 붙여 함께 전송합니다.
  4. HTML 저장 & 시연: 생성된 코드를 복사 → 메모장에 붙여넣기 → 파일명.html로 저장 → 더블클릭하여 브라우저에서 바로 시연합니다.

시나리오: CS 현황 대시보드 즉석 제작

BEST PROMPT "아래 CS 현황 데이터를 기반으로 브라우저에서 바로 실행되는 HTML 대시보드를 만들어 줘. ■ 시각화 요소: 1. 상단: KPI 카드 4개 — 총 접수건, 완료건, 평균 처리일수, CSAT 점수 (각 수치 크게 표시) 2. 중앙: 제품별 CSAT 점수 막대 차트 (색상: LG 레드 계열, 낮은 항목은 강조) 3. 하단: 주차별 접수 추이 꺾은선 그래프 ■ 디자인: - 배경: 흰색, 포인트 컬러: #C10933 (LG 레드) - 폰트: Noto Sans KR - 반응형 레이아웃, 카드 그림자 효과 ■ 기능: 차트에 마우스 오버 시 수치 툴팁 표시 [붙여넣을 데이터] (여기에 엑셀에서 복사한 데이터 붙여넣기)"

시연 가이드: 회의에서 MVP 보여주기

방법 A: 로컬 파일 공유 (가장 빠름)

사내 메신저나 메일로 즉시 공유하는 방법

  1. 1. Copilot이 생성한 코드 전체를 복사(Ctrl+A → Ctrl+C)
  2. 2. 메모장을 열고 붙여넣기 후 파일명.html로 저장
  3. 3. 파일을 더블클릭하면 브라우저에서 즉시 실행 — 회의 중 시연 가능
  4. 4. 팀즈 채팅이나 메일에 html 파일 첨부 공유

MVP 시연의 핵심 가치

  • ✓ 기획 단계에서 "이렇게 보여주면 어떨까요?" 즉석 확인
  • ✓ 회의 참석자 모두가 같은 화면을 보며 피드백 제공 가능
  • ✓ "조금 바꿔줘"를 Copilot에 요청하면 실시간 수정도 가능
  • ✓ 개발팀에 명확한 기준 화면 전달 → 커뮤니케이션 비용 대폭 절감

3-3. HTML 발표자료 제작

3-4. HTML 배포의 의미 & 방법

HTML 배포(Deploy)란 무엇이며 왜 필요할까요?

배포(Deployment)란 로컬 PC에서 혼자 보던 HTML 문서나 애플리케이션 코드를 다른 사람들도 접속해서 사용할 수 있도록 웹상에 띄우고 접근 권한을 부여하는 과정을 의미합니다.

  • 정적 자산 시각화: HTML/CSS로 구성된 대시보드나 발표 자료는 실행 가능한 서버나 경로가 있어야 동적 데이터나 클릭 이벤트가 브라우저에서 정상 작동합니다.
  • 동상이몽 해결: URL 링크 하나 혹은 공유 서버 경로를 통해 팀원이나 경영진이 동일한 결과물을 실시간으로 보며 즉각적으로 소통할 수 있게 합니다.
  • 사내 환경의 대안: 별도의 상용 호스팅이나 복잡한 사내 서버 인프라를 승인받지 않더라도, 간단한 임시/대안 배포 방식을 통해 MVP(최소 기능 제품) 수준의 기획 검증을 현업에서 즉각 진행할 수 있습니다.

Copilot으로 만든 대시보드나 발표자료(HTML)를 팀원, 경영진, 외부 파트너에게 공유하는 3가지 실전 배포 방법을 안내합니다.

방법 A: HTML 파일 직접 전송

⚡ 가장 빠름 (30초)

  1. 1. Copilot이 생성한 코드 전체를 복사
  2. 2. 메모장을 열고 붙여넣기
  3. 3. 파일명.html로 저장
  4. 4. 팀즈 채팅·메일에 파일 첨부 공유
  5. 5. 받은 사람은 더블클릭으로 바로 실행
가장 간편하지만, 파일을 받아야만 볼 수 있는 제한이 있습니다.

방법 B: GitHub Pages 배포

🔗 URL 링크로 공유

  1. 1. github.com 회원가입 후 Public Repository 생성
  2. 2. 저장소에 index.html 파일 업로드
  3. 3. Settings > Pages > Branch: main 선택 후 Save
  4. 4. 생성된 https://username.github.io/repo/ 링크로 공유
인터넷만 되면 누구나 접근 가능. 모바일에서도 열람 가능합니다.

방법 C: 내 PC를 서버로 사용

🖥️ 사내 실시간 시연 최적

  1. 1. HTML 파일을 바탕화면에 저장
  2. 2. Win+R → cmd 입력 후 실행
  3. 3. 아래 명령어 입력:
    cd Desktop && python -m http.server 8080
  4. 4. 브라우저에서 http://localhost:8080 접속
  5. 5. 같은 네트워크 PC에서 내 IP:8080으로도 접근 가능
Python이 없으면 npx serve . 명령어로도 가능합니다. 사내 서버가 없어도 회의 시연에 최적입니다.

4-1. MS Cowork 소개 & 실습

MS Cowork란?

Microsoft Copilot의 에이전트(Agents) 탭에서 선택할 수 있는 협업형 AI 어시스턴트입니다. 개인 업무를 넘어 팀원들과 함께 AI 기반으로 협업하고, 지식을 공유하며 업무 효율을 높이는 데 특화되어 있습니다.

Copilot 채팅 → 상단 "에이전트(Agents)" 탭 클릭 → Cowork 선택

Cowork 핵심 기능

공유 지식베이스

팀 문서, 회의록, 업무 매뉴얼을 업로드하면 팀원 모두가 같은 맥락으로 AI에게 질문할 수 있습니다.

협업 채팅

팀원과 채팅하면서 AI에게 동시에 질문하고, 답변을 함께 확인하며 의사결정을 빠르게 내릴 수 있습니다.

작업 할당 & 추적

AI가 회의 내용을 자동 분석해 액션 아이템을 추출하고, 담당자별로 할당 및 진행 상황을 추적합니다.

📋 시나리오: CS팀 주간 보고 준비

5-1. 조별 실습 과제

지금부터는 실전입니다!

"오늘 배운 것을 본업에 돌아가서 바로 쓸 수 있는 결과물"을 만들어 주세요.

교육 시간 안에 완성하지 못해도 괜찮습니다. 핵심은 직접 손으로 만들어 보는 경험입니다.

👥

3~5명 한 조로 구성

🎯

과제 선택

아래 3가지 중 택 1

🛠️

제작 & 협업

함께 만들고 토의

📢

발표 & 공유

결과물 시연 및 피드백

A

에이전트 만들기

MS Copilot에서 우리 팀 업무에 도움이 되는 맞춤형 지식 FAQ 에이전트를 설계합니다.

예시 과제:

  • BI매트릭스 CS 데이터마트 가이드 챗봇
  • CS 상담 응대 FAQ 및 업무 매뉴얼 챗봇
  • 신규 입사자 전용 부서 온보딩 안내 챗봇
도구: MS Copilot 웹 포털 (에이전트 만들기 기능)
B

Cowork 협업 설계

MS Cowork 에이전트를 활성화하여 팀원들과의 공유 지식망 및 협업 태스크를 시연합니다.

예시 과제:

  • 글로벌 VOC 만족도 요약 보고서 공동 검토
  • 부서 내 지식 자료 연동을 통한 공동 QA 진행
  • 팀 내 협업 일정 및 공용 가이드 북 제작
도구: MS Copilot (Cowork 에이전트 탭)
C

웹페이지(MVP) 만들기

아웃룩 Copilot(Claude)으로 실무에 쓸 수 있는 웹페이지를 제작하고 배포합니다.

예시 과제:

  • CS 실적 대시보드 제작 및 시연
  • 팀 업무 현황 보고용 HTML 슬라이드
  • 인터랙티브 웹 기반 업무 체크리스트
배포: 파일 전송 / GitHub Pages / 내 PC 서버 (3-4 참고)
